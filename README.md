# Bike Store Data Migration & Analysis (Part 2)
## This project focuses on the transition of a relational database from MySQL to SQLite to facilitate local data science workflows and in-depth business analysis using Python.

#üìå Project Overview
- The objective was to take an existing "Bike Store" database from a MySQL server and migrate its entire schema and dataset into an SQLite database file (bike_store.db). This allows for    more flexible analysis using Jupyter Notebooks without the need for a persistent server connection.

# üìÇ Project Structure
- migrate.py: A Python script that handles the connection to MySQL, creates the equivalent schema in SQLite, and copies all records.
- bike_store_queries.ipynb: A Jupyter Notebook containing data analysis, revenue calculations, and visualizations.
- bike_store.db: The resulting SQLite database (generated by the migration script).

# ‚öôÔ∏è Migration Process
## The migration was performed using the migrate.py script. It performs the following steps:
- Connects to a local MySQL instance.
- Maps MySQL data types (e.g., decimal, varchar, int) to compatible SQLite types.
- Automatically detects tables, primary keys, and data to recreate the database structure locally.
- Converts Decimal objects to floats to ensure compatibility with SQLite's storage engine.
